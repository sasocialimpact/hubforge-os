    97‚Üí    97‚Üí### Key design decisions
    98‚Üí    98‚Üí
    99‚Üí    99‚Üí1. **No socket.io** - The frontend orchestrates the reasoning loop via sequential HTTP API calls. This works on Vercel's serverless platform (no persistent connections needed).
   100‚Üí   100‚Üí
   101‚Üí   101‚Üí2. **API keys in browser** - User API keys (OpenAI, Groq, etc.) are stored in localStorage and sent directly to the provider. They never touch HubForge servers.
   102‚Üí   102‚Üí
   103‚Üí   103‚Üí3. **localStorage-first** - All user data (programs, org profile, context blocks, usage) is stored in localStorage. Supabase is optional for persistence across devices.
   104‚Üí   104‚Üí
   105‚Üí   105‚Üí4. **Serverless-friendly** - Each API route is a standalone function with `maxDuration` set. No state between requests.

   210‚Üí   210‚Üí## 3. The 9-Engine Reasoning Pipeline

   309‚Üí   309‚Üí### Engine implementations
   310‚Üí   310‚Üí
   311‚Üí   311‚ÜíAll engines are in `src/lib/engines.ts`. Each engine:
   312‚Üí   312‚Üí
   313‚Üí   313‚Üí1. Takes typed inputs
   314‚Üí   314‚Üí2. Constructs a system prompt + user prompt
   315‚Üí   315‚Üí3. Calls the LLM via the `llm()` function (with timeout + retry)
   316‚Üí   316‚Üí4. Parses the response (JSON extraction for structured outputs)
   317‚Üí   317‚Üí5. Returns typed outputs
   318‚Üí   318‚Üí
   319‚Üí   319‚Üí```typescript
   320‚Üí   320‚Üí// Example: Reasoning Engine signature
   321‚Üí   321‚Üíexport async function reasoningEngine(
   322‚Üí   322‚Üí  config: ProviderConfig,           // AI provider config
   323‚Üí   323‚Üí  problem: string,                  // User's problem text
   324‚Üí   324‚Üí  decomposition: Decomposition,     // From Supervisor
   325‚Üí   325‚Üí  retrieval: RetrievalResult,       // From Retrieval Engine
   326‚Üí   326‚Üí  priorCritique: string | null,     // From previous iteration's Critique
   327‚Üí   327‚Üí  priorDraft: string | null,        // From previous iteration's draft
   328‚Üí   328‚Üí  pack: DomainPack,                 // Knowledge pack
   329‚Üí   329‚Üí  iteration: number,                // Current iteration (1-5)
   330‚Üí   330‚Üí  maxIterations: number,            // Max iterations (configurable)
   331‚Üí   331‚Üí  outputTypes: OutputType[],        // What outputs to produce
   332‚Üí   332‚Üí  answers?: Record<string, string>, // User's interview answers
   333‚Üí   333‚Üí  webSearch?: WebSearchResult,      // Web search results
   334‚Üí   334‚Üí  orgContext?: string,              // Organization context
   335‚Üí   335‚Üí  contextBlocks?: string,           // Reusable context blocks
   336‚Üí   336‚Üí): Promise<string>                  // Returns the draft (Markdown)
   337‚Üí   337‚Üí```
   338‚Üí   338‚Üí
   339‚Üí   339‚Üí### LLM Router
   340‚Üí   340‚Üí
   341‚Üí   341‚ÜíThe `llm()` function routes to the correct provider:
   342‚Üí   342‚Üí
   343‚Üí   343‚Üí```typescript
   344‚Üí   344‚Üí// Z.ai (default, uses SDK)
   345‚Üí   345‚Üíif (config.provider === 'zai') {
   346‚Üí   346‚Üí  const zai = await getZAI()
   347‚Üí   347‚Üí  const completion = await zai.chat.completions.create({ ... })
   348‚Üí   348‚Üí  return completion.choices[0]?.message?.content
   349‚Üí   349‚Üí}
   350‚Üí   350‚Üí
   351‚Üí   351‚Üí// Z.ai with user's own key (OpenAI-compatible endpoint)
   352‚Üí   352‚Üíif (config.provider === 'zai-key') {
   353‚Üí   353‚Üí  // Falls through to OpenAI-compatible fetch
   354‚Üí   354‚Üí}
   355‚Üí   355‚Üí
   356‚Üí   356‚Üí// All other providers (OpenAI, Anthropic, Gemini, Groq, Local)
   357‚Üí   357‚Üíconst res = await fetch(`${baseUrl}/chat/completions`, {
   358‚Üí   358‚Üí  method: 'POST',
   359‚Üí   359‚Üí  headers: { 'Authorization': `Bearer ${apiKey}`, ... },
   360‚Üí   360‚Üí  body: JSON.stringify({ model, messages, temperature }),
   361‚Üí   361‚Üí})
   362‚Üí   362‚Üí```
   363‚Üí   363‚Üí
   364‚Üí   364‚Üí### Timeout + Retry
   365‚Üí   365‚Üí
   366‚Üí   366‚ÜíEvery LLM call has:
   367‚Üí   367‚Üí- **90-second timeout** (configurable in Geek Mode)
   368‚Üí   368‚Üí- **2 retries** with exponential backoff (3s, 6s)
   369‚Üí   369‚Üí- **Empty response check** (throws if LLM returns empty)
   370‚Üí   370‚Üí
   371‚Üí   371‚Üí```typescript
   372‚Üí   372‚Üíconst LLM_TIMEOUT_MS = 90000
   373‚Üí   373‚Üíconst LLM_MAX_RETRIES = 2
   374‚Üí   374‚Üí
   375‚Üí   375‚Üífor (let attempt = 1; attempt <= LLM_MAX_RETRIES + 1; attempt++) {
   376‚Üí   376‚Üí  try {
   377‚Üí   377‚Üí    // Call LLM with timeout
   378‚Üí   378‚Üí    const result = await withTimeout(llmCall, LLM_TIMEOUT_MS)
   379‚Üí   379‚Üí    if (!result.trim()) throw new Error('Empty response')
   380‚Üí   380‚Üí    return result
   381‚Üí   381‚Üí  } catch (e) {
   382‚Üí   382‚Üí    if (attempt <= LLM_MAX_RETRIES) {
   383‚Üí   383‚Üí      await new Promise(r => setTimeout(r, attempt * 3000))
   384‚Üí   384‚Üí    }
   385‚Üí   385‚Üí  }
   386‚Üí   386‚Üí}
   387‚Üí   387‚Üí```
   388‚Üí   388‚Üí
   389‚Üí   389‚Üí---

   391‚Üí   391‚Üí## 4. Knowledge Graph (8 Layers)
   392‚Üí   392‚Üí
   393‚Üí   393‚ÜíThe Social Impact Pack knowledge graph is defined in `src/lib/knowledge.ts`. It has 8 layers that feed different engines:
   394‚Üí   394‚Üí
   395‚Üí   395‚Üí| Layer | Name | Contents | Used by |
   396‚Üí   396‚Üí|-------|------|----------|---------|
   397‚Üí   397‚Üí| 1 | Domain Knowledge | Sectors (Education, Health, WASH, etc.) | Supervisor |
   398‚Üí   398‚Üí| 2 | Framework Knowledge | 6 frameworks (ToC, Logframe, Outcome Mapping, MSC, Impact Eval, Survey Design) | Retrieval Engine |
   399‚Üí   399‚Üí| 3 | Procedural Knowledge | Program design process, evaluation process | Supervisor |
   400‚Üí   400‚Üí| 4 | Decision Rules | 5 deterministic rules (SMART, Stakeholders, Assumptions, Evidence, Risk) | Rule Engine |
   401‚Üí   401‚Üí| 5 | Evidence Libraries | 5 evidence sources (OECD-DAC, Better Evaluation, Innosight, World Bank, CGAP) | Reasoning Engine |
   402‚Üí   402‚Üí| 6 | Historical Memory | 3 past cases (farmer livelihoods, girls' education, rural healthcare) | Reasoning Engine |
   403‚Üí   403‚Üí| 7 | Reasoning Patterns | 6 patterns (Root Cause, Counterfactual, Tradeoff, Risk, Comparative, Contribution) | Reasoning Engine |
   404‚Üí   404‚Üí| 8 | Improvement Heuristics | 6 heuristics (Weak assumptions, Missing evidence, Vague‚Üímeasurable, Causal logic, Inconsistency, Reduce uncertainty) | Critique Engine |

   828‚Üí   828‚Üítype ProviderId = 'zai' | 'zai-key' | 'openai' | 'anthropic' | 'gemini' | 'groq' | 'local'

   833‚Üí   833‚Üíinterface ProviderConfig {

   835‚Üí   835‚Üí  apiKey?: string    // stored in browser localStorage only

   841‚Üí   841‚Üí### How keys work
   842‚Üí   842‚Üí
   843‚Üí   843‚Üí1. User selects provider in Settings dialog
   844‚Üí   844‚Üí2. If provider needs key, API key field appears
   845‚Üí   845‚Üí3. Key stored in `localStorage` under `hubforge.providerConfig`
   846‚Üí   846‚Üí4. Key sent directly to provider with each API call (via fetch)
   847‚Üí   847‚Üí5. Key NEVER sent to HubForge servers
   848‚Üí   848‚Üí
   849‚Üí   849‚Üí### Adding a new provider
   850‚Üí   850‚Üí
   851‚Üí   851‚Üí1. Add to `ProviderId` type in `providers.ts` and `engines.ts`
   852‚Üí   852‚Üí2. Add to `DEFAULT_BASE_URLS` and `DEFAULT_MODELS` in `engines.ts`
   853‚Üí   853‚Üí3. Add to `PROVIDER_LABELS` in `engines.ts`
   854‚Üí   854‚Üí4. Add to `PROVIDERS` array in `providers.ts` with metadata
   855‚Üí   855‚Üí5. If the provider is OpenAI-compatible, it works automatically via the `fetch()` path in `llm()`
   856‚Üí   856‚Üí6. If not OpenAI-compatible, add custom routing in `llm()`
   857‚Üí   857‚Üí

   860‚Üí   860‚Üí## 10. Data Flow: End-to-End
   861‚Üí   861‚Üí
   862‚Üí   862‚Üí```
   863‚Üí   863‚Üí1. User types problem in General Mode
   864‚Üí   864‚Üí   ‚Üí stored in `problem` state
   865‚Üí   865‚Üí
   866‚Üí   866‚Üí2. User clicks "Help me build it"
   867‚Üí   867‚Üí   ‚Üí POST /api/interview (Supervisor Engine)
   868‚Üí   868‚Üí   ‚Üí Returns decomposition + clarifying questions
   869‚Üí   869‚Üí
   870‚Üí   870‚Üí3. User answers questions (or skips)
   871‚Üí   871‚Üí   ‚Üí Answers stored in `answers` state
   872‚Üí   872‚Üí
   873‚Üí   873‚Üí4. User clicks "Build my strategy + ToC"
   874‚Üí   874‚Üí   ‚Üí runLoop() starts
   875‚Üí   875‚Üí
   876‚Üí   876‚Üí5. runLoop() calls sequentially:
   877‚Üí   877‚Üí   a. POST /api/run-step { step: 'retrieval' }
   878‚Üí   878‚Üí      ‚Üí Returns frameworks, rules, evidence from knowledge graph
   879‚Üí   879‚Üí   
   880‚Üí   880‚Üí   b. POST /api/search
   881‚Üí   881‚Üí      ‚Üí Returns demographics, previous programs, evidence from web
   882‚Üí   882‚Üí      ‚Üí Auto-saved as context block
   883‚Üí   883‚Üí   
   884‚Üí   884‚Üí   c. For each iteration (1 to maxIterations):
   885‚Üí   885‚Üí      i.   POST /api/run-step { step: 'rule' }
   886‚Üí   886‚Üí      ii.  POST /api/run-step { step: 'reasoning' }
   887‚Üí   887‚Üí           ‚Üí Includes: orgContext, webSearch, retrieval, answers
   888‚Üí   888‚Üí      iii. POST /api/run-step { step: 'critique' }
   889‚Üí   889‚Üí      iv.  POST /api/run-step { step: 'improvement' }
   890‚Üí   890‚Üí      v.   POST /api/run-step { step: 'evaluation' }
   891‚Üí   891‚Üí      vi.  If score >= threshold: break
   892‚Üí   892‚Üí   
   893‚Üí   893‚Üí   d. POST /api/structure
   894‚Üí   894‚Üí      ‚Üí Returns ToC JSON + Logframe JSON
   895‚Üí   895‚Üí
   896‚Üí   896‚Üí6. Deliverable shown to user
   897‚Üí   897‚Üí   ‚Üí Strategy (Markdown, rendered)
   898‚Üí   898‚Üí   ‚Üí ToC (visual diagram, editable)
   899‚Üí   899‚Üí   ‚Üí Logframe (table, editable)
   900‚Üí   900‚Üí
   901‚Üí   901‚Üí7. Auto-save:
   902‚Üí   902‚Üí   ‚Üí Program saved to localStorage (programs.ts)
   903‚Üí   903‚Üí   ‚Üí Session saved to /api/memory (if Supabase configured)
   904‚Üí   904‚Üí   ‚Üí Web search saved as context block (context-blocks.ts)
   905‚Üí   905‚Üí
   906‚Üí   906‚Üí8. User can:
   907‚Üí   907‚Üí   ‚Üí Edit ToC/Logframe inline
   908‚Üí   908‚Üí   ‚Üí Export to Word/PDF/Excel
   909‚Üí   909‚Üí   ‚Üí Give feedback ‚Üí POST /api/feedback ‚Üí revised draft
   910‚Üí   910‚Üí
   911‚Üí   911‚Üí9. Every API call tracked:
   912‚Üí   912‚Üí   ‚Üí usage-tracker.ts records provider, engine, duration, tokens
   913‚Üí   913‚Üí   ‚Üí Usage panel shows consumption + optimization tips
   914‚Üí   914‚Üí
   915‚Üí   915‚Üí10. Every user action tracked:
   916‚Üí   916‚Üí    ‚Üí analytics.ts records events (app_open, run_start, etc.)
   917‚Üí   917‚Üí    ‚Üí Admin dashboard shows funnel, scores, errors
   918‚Üí   918‚Üí```
   919‚Üí   919‚Üí

   927‚Üí   927‚Üí-- Reasoning sessions
   928‚Üí   928‚ÜíCREATE TABLE reasoning_sessions (
   929‚Üí   929‚Üí  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,

   933‚Üí   933‚Üí  iterations INTEGER DEFAULT 0,
   934‚Üí   934‚Üí  final_score INTEGER DEFAULT 0,
   935‚Üí   935‚Üí  threshold_met BOOLEAN DEFAULT FALSE,
   936‚Üí   936‚Üí  final_draft TEXT,
   937‚Üí   937‚Üí  structured_outputs JSONB,

  1081‚Üí  1081‚Üí## 16. Security Model
  1082‚Üí  1082‚Üí
  1083‚Üí  1083‚Üí### API Key Storage
  1084‚Üí  1084‚Üí- User API keys stored in browser `localStorage` only
  1085‚Üí  1085‚Üí- Keys sent directly to AI provider (OpenAI, Groq, etc.) via `fetch()`
  1086‚Üí  1086‚Üí- Keys NEVER sent to HubForge OS servers
  1087‚Üí  1087‚Üí- Keys NEVER logged to console
  1088‚Üí  1088‚Üí
  1089‚Üí  1089‚Üí### Input Validation
  1090‚Üí  1090‚ÜíAll API routes validate inputs:
  1091‚Üí  1091‚Üí- `problem`: required, string, max 10000 chars
  1092‚Üí  1092‚Üí- `feedback`: required, max 5000 chars
  1093‚Üí  1093‚Üí- `eventType`: alphanumeric + underscore only, max 100 chars
  1094‚Üí  1094‚Üí- All analytics fields: sanitized with `.slice()` length limits
  1095‚Üí  1095‚Üí- `user_agent` and `referrer`: truncated to 500 chars
  1096‚Üí  1096‚Üí
  1097‚Üí  1097‚Üí### Admin Protection
  1098‚Üí  1098‚Üí- Admin endpoints require `admin_key` query parameter
  1099‚Üí  1099‚Üí- Default key: `hubforge-admin-2024` (change via `HUBFORGE_ADMIN_KEY` env var)
  1100‚Üí  1100‚Üí- Wrong key returns 403
  1101‚Üí  1101‚Üí
  1102‚Üí  1102‚Üí### Rate Limiting
  1103‚Üí  1103‚Üí- Not implemented (Vercel handles per-IP limits)
  1104‚Üí  1104‚Üí- `maxDuration` on all routes prevents infinite execution
  1105‚Üí  1105‚Üí
  1106‚Üí  1106‚Üí### Data Encryption
  1107‚Üí  1107‚Üí- HTTPS/TLS 1.3 in transit (Vercel default)
  1108‚Üí  1108‚Üí- Supabase: AES-256 at rest
  1109‚Üí  1109‚Üí- localStorage: not encrypted (browser security model)
  1110‚Üí  1110‚Üí
